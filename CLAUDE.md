# CLAUDE.md - 自习室预约管理系统开发指南

## 项目概述

本项目使用纯C语言实现一个自习室预约管理系统，用于图书馆自习室座位的预约、签到、超时管理等功能。

## 开发环境

- **语言**：C语言（C99/C11标准）
- **编译器**：GCC（推荐）或 MSVC
- **操作系统**：Windows/Linux/MacOS
- **依赖**：无第三方库依赖，仅使用标准库

## 开发规范

### 1. 命名规范

- **文件名**：小写字母，单词间用下划线分隔（如 `student_manager.c`）
- **函数名**：大驼峰命名法（如 `CreateReservation`）
- **变量名**：小驼峰命名法（如 `studentId`）
- **常量/宏**：全大写，下划线分隔（如 `MAX_STUDENT_ID_LEN`）
- **类型名**：大驼峰命名法（如 `StudentHashTable`）
- **枚举值**：全大写，下划线分隔（如 `SEAT_AVAILABLE`）

### 2. 代码风格

- 使用4个空格缩进，不使用Tab
- 大括号换行（Allman风格）
- 函数之间空一行
- 关键逻辑添加注释
- 每个函数不超过50行（复杂算法除外）

### 3. 注释规范

```c
/**
 * @brief 函数功能简述
 * @param paramName 参数说明
 * @return 返回值说明
 * @note 注意事项
 */
```

### 4. 错误处理

- 所有公开函数返回 `ErrorCode` 枚举
- 内存分配失败立即终止程序
- 参数校验在函数入口完成
- 错误信息通过日志记录

### 5. 内存管理

- 使用 `safeMalloc` 和 `safeRealloc` 进行内存分配
- 每个 `malloc` 必须有对应的 `free`
- 链表销毁时遍历释放所有节点
- 程序退出前释放所有资源

## 开发步骤

### 阶段1：基础数据结构实现

1. **实现链表**（`linkedlist.c`）
   - 创建节点、插入、删除、查找、销毁
   - 支持学生、座位、预约三种链表

2. **实现哈希表**（`hashtable.c`）
   - 哈希函数设计
   - 插入、查找、删除
   - 冲突处理（链地址法）

3. **实现栈**（`stack.c`）
   - 入栈、出栈、判空
   - 用于撤销操作

4. **实现队列**（`queue.c`）
   - 入队、出队、判空
   - 用于等待队列

5. **实现最小堆**（`heap.c`）
   - 插入、删除堆顶、调整
   - 用于超时事件管理

### 阶段2：核心业务模块

1. **学生管理模块**（`student.c`）
   - 注册、登录、信息查询、信用分管理

2. **座位管理模块**（`seat.c`）
   - 查询可用座位、状态更新、推荐座位

3. **预约管理模块**（`reservation.c`）
   - 创建预约、取消预约、签到签退
   - 冲突检测、过期处理

4. **超时管理模块**（`timeout.c`）
   - 空闲检测、超时处理、自动释放

### 阶段3：辅助功能模块

1. **统计分析模块**（`statistics.c`）
   - 使用率、预约率、高峰时段统计

2. **系统管理模块**（`system.c`）
   - 自习室管理、座位布局、数据持久化

3. **工具函数**（`utils.c`）
   - 时间处理、ID生成、日志记录

### 阶段4：主程序与测试

1. **主程序**（`main.c`）
   - 菜单界面、用户交互
   - 系统初始化与销毁

2. **单元测试**（`tests/`）
   - 每个模块的测试用例
   - 边界条件测试

## 关键算法

### 1. 哈希函数
使用BKDR哈希算法，保证良好的散列效果。

### 2. 冲突检测
遍历座位预约记录，检查时间区间是否重叠。

### 3. 超时检测
使用最小堆维护超时事件，O(1)获取最早超时事件。

### 4. 链表排序
使用归并排序，保证O(n log n)时间复杂度。

## 注意事项

1. **内存安全**
   - 所有字符串操作使用安全函数（`strncpy` 而非 `strcpy`）
   - 数组访问前检查边界
   - 避免内存泄漏

2. **线程安全**
   - 当前版本为单线程实现
   - 如需多线程，需添加互斥锁

3. **数据持久化**
   - 使用文件存储数据
   - 定期备份重要数据

4. **性能考虑**
   - 哈希表大小选择素数
   - 避免不必要的内存分配
   - 批量操作减少IO

5. **可移植性**
   - 使用标准C库函数
   - 避免平台特定API
   - 时间处理使用 `time.h`

## 常见问题

### Q: 如何处理大量预约？
A: 使用哈希表快速查找，链表存储预约记录，优先队列管理超时事件。

### Q: 如何保证数据一致性？
A: 操作前校验状态，操作后更新所有相关数据结构，使用事务机制。

### Q: 如何扩展支持多个自习室？
A: 当前设计已支持，通过roomId区分不同自习室。
