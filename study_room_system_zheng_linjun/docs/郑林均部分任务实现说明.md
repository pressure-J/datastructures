# 郑林均部分任务实现说明

## 一、问题分析与理解

郑林均负责自习室预约管理系统的基础数据支撑部分，包括链表、哈希表、学生管理、座位管理、自习室和座位基础数据维护。预约、签到、签退、超时检测等后续功能都要依赖本模块提供的查询和状态维护接口。

## 二、数据结构设计（C语言）

### 2.1 枚举类型

在 `include/types.h` 中定义座位状态 `SeatStatus`、预约状态 `ReservationStatus` 和错误码 `ErrorCode`。

### 2.2 核心结构体

在 `include/types.h` 中统一定义：

- `Student`：学生信息
- `StudyRoom`：自习室信息
- `Seat`：座位信息
- `Reservation`：预约记录

### 2.3 哈希表结构

学生、座位、预约分别使用哈希表保存：

- `StudentHashTable`
- `SeatHashTable`
- `ReservationHashTable`

哈希冲突采用链地址法处理。

## 五、关键算法设计（C语言实现）

### 5.1 BKDR 哈希算法

使用 `seed = 131` 的 BKDR 哈希算法，把学号、座位编号、预约编号映射到哈希表下标。

### 5.2 链地址法

每个哈希桶保存一个链表。插入时采用头插法，查找时遍历对应桶链表。

### 5.3 时间冲突检测

判断两个时间区间是否冲突：

```c
startTime < res->endTime && endTime > res->startTime
```

如果预约状态为待签到或已签到，则认为该时间段会占用座位。

## 六、数据结构选择理由

| 数据结构 | 用途 | 选择理由 |
|---------|------|---------|
| 链表 | 保存动态数据和查询结果 | 插入删除方便 |
| 哈希表 | 学生、座位、预约快速查找 | 平均 O(1) 查询 |
| 二维数组 | 座位布局 | 行列结构直观 |
| 枚举 | 状态管理 | 状态含义清楚 |

## 七、内存管理策略

所有动态申请的链表、哈希节点和座位布局都提供对应释放函数。查询可用座位返回的链表由调用者使用 `destroySeatList` 释放。

## 八、异常处理设计

公开业务函数返回 `ErrorCode`。学生不存在返回 `ERR_STUDENT_NOT_FOUND`，座位不存在返回 `ERR_SEAT_NOT_FOUND`，时间不合法返回 `ERR_INVALID_TIME`，内存申请失败返回 `ERR_MEMORY_ALLOC`。

## 九、性能优化策略

1. 使用哈希表提升编号查询效率。
2. 使用链地址法降低冲突处理复杂度。
3. 查询可用座位时只复制符合条件的座位。
4. 座位布局先计算最大行列，再创建二维数组。

## 十一、测试说明

