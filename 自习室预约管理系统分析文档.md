# 自习室预约管理系统分析文档（C语言版）

## 一、系统概述

### 1.1 系统背景
随着高校图书馆自习室使用需求的增加，传统的人工管理方式已无法满足高效管理的需求。特别是在疫情防控期间，需要实现预约制管理，确保1人1桌1座，避免人员聚集。本系统旨在通过信息化手段，实现自习室座位的预约、签到、签退、超时管理等功能，提高座位使用效率。

### 1.2 系统目标
- 实现自习室座位的在线预约管理
- 提供签到签退功能，确保预约有效性
- 自动检测空闲超时，释放未使用座位
- 提供数据统计分析功能，优化资源分配
- 保障数据一致性和系统稳定性

### 1.3 系统范围
本系统涵盖以下核心功能：
- 学生信息管理
- 自习室与座位管理
- 预约创建与取消
- 签到与签退处理
- 超时检测与处理
- 使用情况统计分析

## 二、需求分析

### 2.1 功能需求

#### 2.1.1 学生管理需求
| 需求编号 | 需求描述 | 优先级 |
|---------|---------|-------|
| FR-001 | 学生注册：输入学号、姓名、联系方式等信息完成注册 | 高 |
| FR-002 | 学生登录：通过学号验证身份 | 高 |
| FR-003 | 信息查询：查看个人基本信息及信用分 | 中 |
| FR-004 | 信息更新：修改联系方式等个人信息 | 中 |
| FR-005 | 信用分管理：违约时扣除信用分，信用分不足时限制预约 | 高 |

#### 2.1.2 座位管理需求
| 需求编号 | 需求描述 | 优先级 |
|---------|---------|-------|
| FR-006 | 座位查询：按自习室、时间段查询可用座位 | 高 |
| FR-007 | 座位状态查看：查看座位当前状态（可用/已预约/使用中/维护） | 高 |
| FR-008 | 座位推荐：根据历史数据推荐座位 | 低 |
| FR-009 | 座位布局查看：查看自习室座位分布图 | 中 |

#### 2.1.3 预约管理需求
| 需求编号 | 需求描述 | 优先级 |
|---------|---------|-------|
| FR-010 | 创建预约：选择自习室、座位、时间段完成预约 | 高 |
| FR-011 | 取消预约：在规定时间内取消预约 | 高 |
| FR-012 | 签到：预约后在规定时间内扫码签到 | 高 |
| FR-013 | 签退：使用完毕后签退释放座位 | 高 |
| FR-014 | 预约查询：查看个人预约历史记录 | 中 |
| FR-015 | 冲突检测：自动检测时间冲突，防止重复预约 | 高 |

#### 2.1.4 超时管理需求
| 需求编号 | 需求描述 | 优先级 |
|---------|---------|-------|
| FR-016 | 签到超时检测：预约后未在规定时间内签到自动取消 | 高 |
| FR-017 | 空闲超时检测：签到后空闲超过30分钟自动释放座位 | 高 |
| FR-018 | 超时处理：超时后自动更新状态并扣除信用分 | 高 |

#### 2.1.5 统计分析需求
| 需求编号 | 需求描述 | 优先级 |
|---------|---------|-------|
| FR-019 | 使用率统计：计算自习室/座位使用率 | 中 |
| FR-020 | 预约率统计：统计预约成功率、取消率等 | 中 |
| FR-021 | 高峰时段分析：识别使用高峰时段 | 低 |
| FR-022 | 违约统计：统计学生违约情况 | 中 |
| FR-023 | 热门座位排行：统计最受欢迎的座位 | 低 |

### 4.2 数据字典

#### 4.2.1 学生信息
| 数据项 | 类型 | 长度 | 说明 |
|-------|------|------|------|
| studentId | 字符串 | 20 | 学号，唯一标识 |
| name | 字符串 | 50 | 学生姓名 |
| phone | 字符串 | 15 | 联系电话 |
| department | 字符串 | 100 | 所属院系 |
| creditScore | 整数 | - | 信用分，初始100 |
| registerTime | 时间戳 | - | 注册时间 |

#### 4.2.2 座位信息
| 数据项 | 类型 | 长度 | 说明 |
|-------|------|------|------|
| seatId | 字符串 | 30 | 座位编号，如"A101-001" |

| row | 整数 | - | 座位排号 |
| column | 整数 | - | 座位列号 |
| status | 枚举 | - | 座位状态 |
| currentStudent | 字符串 | 20 | 当前使用学生学号 |
| lastActiveTime | 时间戳 | - | 最后活跃时间 |

#### 4.2.3 预约信息
| 数据项 | 类型 | 长度 | 说明 |
|-------|------|------|------|
| reservationId | 字符串 | 30 | 预约编号，唯一标识 |
| studentId | 字符串 | 20 | 学生学号 |
| seatId | 字符串 | 30 | 座位编号 |
| roomId | 字符串 | 20 | 自习室编号 |
| reserveTime | 时间戳 | - | 预约创建时间 |
| startTime | 时间戳 | - | 预约开始时间 |
| endTime | 时间戳 | - | 预约结束时间 |
| checkInTime | 时间戳 | - | 实际签到时间 |
| checkOutTime | 时间戳 | - | 实际签退时间 |
| status | 枚举 | - | 预约状态 |
| cancelReason | 字符串 | 200 | 取消原因 |
