# 自习室预约管理系统设计文档

## 一、问题分析与理解
### 1 问题描述
### 1.1 业务场景
某校图书馆向全体学生免费开放自习阅览室，由于疫情管控需要，要求：
- 学生每日预约，凭预约记录（扫码）进出
- 严格进行1人1桌1座管理
- 空闲时间超过一定时长即预约失效
- 提高座位使用效率

### 1.2 核心需求
1. **预约管理**：学生可预约未来时间的座位
2. **签到管理**：预约后需在规定时间内扫码签到
3. **座位管理**：实时监控座位使用状态
4. **超时失效**：签到后空闲超过阈值自动释放座位
5. **数据统计**：使用率、预约率等统计

### 1.3 约束条件
- 同一时间段，一个座位只能被一个学生预约
- 同一学生同一时间只能预约一个座位
- 预约需在每日开放时间内
- 签到后空闲超过30分钟自动取消预约

### 3.2 核心功能详细设计

#### 3.2.1 预约功能
**输入**：学生学号、自习室编号、座位编号、预约时间段
**处理流程**：
1. 验证学生身份和信用分
2. 检查座位是否可用
3. 检查时间段是否冲突
4. 检查学生是否有其他预约冲突
5. 创建预约记录
6. 更新座位状态为"已预约"
7. 返回预约成功信息

**时间复杂度**：O(n) - 链表遍历查找冲突

#### 3.2.2 签到功能
**输入**：预约编号、学生学号
**处理流程**：
1. 验证预约记录存在且状态为"待签到"
2. 验证当前时间在预约时间内
3. 更新预约状态为"已签到"
4. 更新座位状态为"使用中"
5. 记录签到时间
6. 启动空闲计时器

**时间复杂度**：O(1) - 哈希表查找

#### 3.2.3 签退功能
**输入**：预约编号
**处理流程**：
1. 验证预约记录存在且状态为"已签到"
2. 更新预约状态为"已完成"
3. 更新座位状态为"可用"
4. 记录签退时间
5. 停止空闲计时器

**时间复杂度**：O(1)

#### 3.2.4 超时检测功能
**处理逻辑**：
1. 定期检查所有"已签到"状态的预约
2. 计算当前时间与最后活跃时间的差值
3. 若差值超过阈值（30分钟），执行超时处理
4. 更新预约状态为"超时失效"
5. 释放座位
6. 扣除学生信用分

**时间复杂度**：O(n) - 遍历所有已签到预约

#### 3.2.5 座位查询功能
**输入**：自习室编号、时间范围
**输出**：该时间段内可用座位列表
**处理逻辑**：
1. 获取指定自习室的所有座位
2. 查询每个座位在指定时间段的预约情况
3. 筛选出无预约冲突的座位
4. 返回可用座位列表

**时间复杂度**：O(m * n) - m个座位，每个座位n次查找

### 5.1 哈希函数
```c
/* BKDR哈希函数 */
unsigned int hashString(const char *key) {
    unsigned int seed = 131;  /* 31 131 1313 13131 131313 etc.. */
    unsigned int hash = 0;
    
    while (*key) {
        hash = hash * seed + (*key++);
    }
    
    return (hash & 0x7FFFFFFF) % HASH_TABLE_SIZE;
}
```

### 5.2 冲突检测算法
```c
/* 检查时间区间冲突 */
int hasTimeConflict(ReservationHashTable *table,
                    const char *seatId,
                    time_t startTime, time_t endTime) {
    /* 遍历该座位的所有预约 */
    for (int i = 0; i < HASH_TABLE_SIZE; i++) {
        ReservationHashNode *node = table->buckets[i];
        while (node != NULL) {
            if (strcmp(node->value->seatId, seatId) == 0 &&
                node->value->status != RES_CANCELLED &&
                node->value->status != RES_COMPLETED) {
                /* 检查时间是否重叠 */
                if (startTime < node->value->endTime && 
                    endTime > node->value->startTime) {
                    return 1;  /* 有冲突 */
                }
            }
            node = node->next;
        }
    }
    return 0;  /* 无冲突 */
}
```

### 5.3 最小堆调整算法
```c
/* 上浮调整 */
void heapifyUp(MinHeap *heap, int index) {
    while (index > 0) {
        int parent = (index - 1) / 2;
        if (heap->events[index].timeout < heap->events[parent].timeout) {
            /* 交换 */
            TimeoutEvent temp = heap->events[index];
            heap->events[index] = heap->events[parent];
            heap->events[parent] = temp;
            index = parent;
        } else {
            break;
        }
    }
}

/* 下沉调整 */
void heapifyDown(MinHeap *heap, int index) {
    int smallest = index;
    int left = 2 * index + 1;
    int right = 2 * index + 2;
    
    if (left < heap->size && 
        heap->events[left].timeout < heap->events[smallest].timeout) {
        smallest = left;
    }
    
    if (right < heap->size && 
        heap->events[right].timeout < heap->events[smallest].timeout) {
        smallest = right;
    }
    
    if (smallest != index) {
        /* 交换 */
        TimeoutEvent temp = heap->events[index];
        heap->events[index] = heap->events[smallest];
        heap->events[smallest] = temp;
        heapifyDown(heap, smallest);
    }
}
```

### 5.4 链表排序算法（归并排序）
```c
/* 链表归并排序 */
Reservation* mergeSortReservation(Reservation *head) {
    if (head == NULL || head->next == NULL) {
        return head;
    }
    
    /* 找中点 */
    Reservation *slow = head;
    Reservation *fast = head->next;
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
    }
    
    Reservation *mid = slow->next;
    slow->next = NULL;
    
    /* 递归排序 */
    Reservation *left = mergeSortReservation(head);
    Reservation *right = mergeSortReservation(mid);
    
    /* 合并 */
    return mergeReservationLists(left, right);
}

/* 合并两个有序链表 */
Reservation* mergeReservationLists(Reservation *a, Reservation *b) {
    if (a == NULL) return b;
    if (b == NULL) return a;
    
    Reservation *result = NULL;
    if (a->startTime <= b->startTime) {
        result = a;
        result->next = mergeReservationLists(a->next, b);
    } else {
        result = b;
        result->next = mergeReservationLists(a, b->next);
    }
    return result;
}
```

5. 实现等待队列，当座位暂时不可用时，可以让学生进入等待队列，并按先来先服务原则处理。
6. 实现统计分析模块，包括使用率、预约率、高峰时段、违约统计、座位热度排行和统计报告。
7. 负责设计测试用例，对正常预约、重复预约、时间冲突、取消预约、签到超时、空闲超时等情况进行测试。

**应该怎么做：**

1. 创建预约前，先调用学生查询接口确认学生存在，再调用座位查询接口确认座位存在且可用。
2. 使用 `hasTimeConflict` 检查座位时间冲突，使用 `hasStudentTimeConflict` 检查学生时间冲突。
3. 创建预约成功后，要同时更新预约表和座位状态，保证数据一致。
4. 签到时只允许 `RES_PENDING` 状态的预约签到，签到成功后将预约状态改为 `RES_CHECKED_IN`，座位状态改为 `SEAT_OCCUPIED`。
5. 签退时只允许 `RES_CHECKED_IN` 状态的预约签退，签退成功后将预约状态改为 `RES_COMPLETED`，座位状态改为 `SEAT_AVAILABLE`。
6. 超时检测可以使用最小堆保存超时事件，每次优先处理最早到期的预约，提高处理效率。
7. 统计分析时遍历预约哈希表，根据预约状态、时间范围和自习室编号进行分类统计。
8. 测试时要记录输入、预期输出和实际输出，方便最终写入报告。

### 11.6 模块衔接要求

1. 所有成员必须使用统一的头文件，不能在自己的 `.c` 文件中重复定义结构体。
2. 所有公开函数都要返回 `ErrorCode` 或明确的状态值，方便主程序判断是否执行成功。
3. 修改学生、座位、预约状态时，必须同步更新相关数据，避免出现“预约已完成但座位仍显示占用”的情况。
4. 合并代码前，每个人要先保证自己负责的模块可以单独编译通过。
5. 测试数据格式要统一，学生编号、座位编号、预约编号要符合设计文档中的长度限制。
6. 如果需要修改公共结构体或函数参数，必须先通知另外两名成员，避免其他模块调用失败。
